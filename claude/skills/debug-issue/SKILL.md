# Skill: debug-issue

Use when:
- something is broken or incorrect

Steps:
1. Reproduce the issue
2. Identify root cause (not symptoms)
3. Locate minimal fix
4. Implement fix
5. Validate:
   - issue resolved
   - no regressions introduced

Rules:
- do not guess fixes
- avoid broad changes
- fix root cause only

Definition of Done:
- [ ] issue is reproducible before fix
- [ ] root cause is clearly identified
- [ ] fix is minimal and targeted
- [ ] original issue no longer occurs
- [ ] no new issues or regressions introduced
