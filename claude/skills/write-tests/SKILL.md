# Skill: write-tests

Use when:
- adding or updating tests

Steps:
1. Identify logic being tested
2. Write tests for:
   - expected behavior
   - edge cases
   - failure cases
3. For AI outputs:
   - validate schema
   - validate no hallucination
4. Keep tests minimal and focused

Rules:
- do not over-test trivial code
- prioritize critical logic paths

Definition of Done:
- [ ] tests cover main behavior and edge cases
- [ ] failure cases are included where relevant
- [ ] AI outputs are schema-validated
- [ ] tests are minimal but sufficient
- [ ] tests are readable and focused
